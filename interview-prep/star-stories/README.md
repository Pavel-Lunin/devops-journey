# STAR Stories

STAR-методология (Situation, Task, Action, Result) для behavioural-вопросов на DevOps-интервью.

## Структура

Каждая story — отдельный markdown-файл по шаблону:

- **Situation** — контекст, командa, продукт (обезличенно)
- **Task** — что нужно было решить, какие constraints
- **Action** — что конкретно я сделал, какие решения принял
- **Result** — измеримый итог, что выучил

## Темы, которые точно понадобятся

- Mobile CI/CD pipeline improvements в Сбере (proxy-источник DevOps-перехода)
- Production-проблемы и post-mortem'ы из mobile-опыта
- Принятие решений с trade-off'ами (когда чему-то сказал "нет")
- Конфликты в команде и как разрешали
- "Расскажи о провале" — обязательная категория, готовлю заранее
